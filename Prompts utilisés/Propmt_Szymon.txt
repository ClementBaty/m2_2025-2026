# Prompt — Group 4: File Type Classifier (PyQt5 + KNN)

---

## Context

This is part of a university group project. Six groups collaborate to build a program
that identifies an unknown file's type from its raw signal data.

**Group 4's role:** receive a processed `features.csv` file from Group 3 (which
extracts signal features from the raw file), classify it using KNN against a local
reference database, display the result in a GUI, and export a result CSV.

---

## Tech Stack

- **Language:** Python 3
- **GUI:** PyQt5
- **Data handling:** pandas, numpy, scikit-learn (KNeighborsClassifier or manual KNN)
- **No other major dependencies**

---

## Folder Structure (auto-created on first run if missing)

```
main.py                     ← root of the application
SAMPLE/
    features.csv            ← default input file to classify
DATABASE/
    AUDIO/                  ← example category subfolder
    VIDEO/
    NOISE/
    reference.csv           ← single global reference file, auto-rebuilt
```

- All folders (`SAMPLE/`, `DATABASE/`, and any category subfolders like `AUDIO/`)
  are created automatically when `main.py` is first executed if they do not exist.
- `DATABASE/` contains at most **MAX_CATEGORIES = 7** category subfolders
  (this constant must be defined at the top of the file, easy to change).
- Each category subfolder stores indexed copies of loaded `features.csv` files:
  `1features.csv`, `2features.csv`, `3features.csv`, … (index is a sequential
  integer, no gaps required, just unique).
- `DATABASE/reference.csv` is a single flat CSV rebuilt every time a new file is
  loaded. It aggregates all rows from all category subfolders, adding a `label`
  column whose value is the name of the category subfolder (e.g. `AUDIO`).

---

## Feature Columns Configuration

At the top of `main.py`, define a list:

```python
FEATURE_COLUMNS = [
    "fft_magnitude_0", "fft_magnitude_1", "fft_magnitude_2",
    "fft_frequency_0", "fft_frequency_1", "fft_frequency_2",
    "spectral_centroid", "mean", "std_dev", "kurtosis",
    "skewness", "rms",
    "peak_index_0", "peak_index_1", "peak_index_2",
    "peak_amplitude_0", "peak_amplitude_1", "peak_amplitude_2"
]
```

This list is the **single source of truth** for which columns are used in KNN
comparisons and for building the reference. If Group 3 changes their output
columns, only this list needs updating. `sample_id` is always excluded from
computation (it is a row identifier only).

---

## Reference File (`DATABASE/reference.csv`)

Structure:
```
label, <all columns from FEATURE_COLUMNS>
```

- `label` is the category name (folder name, uppercase).
- Built by scanning all category subfolders in `DATABASE/`, reading every
  indexed `features.csv` file found, appending a `label` column, and
  concatenating all rows.
- Rebuilt from scratch every time a new file is loaded via the Database tab.
- If no reference data exists yet, the reference file is empty (header only).

---

## KNN Classification Logic

- **Distance metric:** Euclidean distance over all `FEATURE_COLUMNS` present in
  both the sample and the reference (columns missing from either side are dropped
  gracefully with a warning print).
- **K formula:**
  ```
  K = 5 + floor(total_database_files / 10)
  ```
  where `total_database_files` is the count of all indexed `features.csv` files
  across all category subfolders.
- **Minimum K = 1.** If fewer than K files exist in the database, use however
  many exist (no crash). This will reduce confidence naturally.
- **Confidence** = proportion of the K nearest neighbors belonging to the
  winning label. Example: if K=5 and 3 neighbors are `AUDIO`, confidence = 0.60.
- The sample to classify is the **first row** of the loaded `features.csv` file.
  If the file has multiple rows, only the first is used (print a warning).
- Normalize all feature columns with StandardScaler fitted on the reference data
  before computing distances.

---

## Export File (`processed_data.csv`)

Saved to the root of the application (same folder as `main.py`).

Columns:
```
sample_id, label, confidence, mean, std_dev, kurtosis, skewness, rms,
dominant_frequency_hz, energy
```

- `sample_id`: taken directly from the `sample_id` column of the input
  `features.csv`. If the column is absent, use the filename as the id.
- `label`: classification result (e.g. `AUDIO`).
- `confidence`: float rounded to 2 decimal places (0.00–1.00).
- `mean`, `std_dev`, `kurtosis`, `skewness`, `rms`: copied directly from
  the input file if present; `NaN` if absent.
- `dominant_frequency_hz`: the `fft_frequency_X` value (X = 0, 1, or 2)
  corresponding to the highest `fft_magnitude_X` in the same row. If FFT
  columns are absent, `NaN`.
- `energy`: computed as `sum(fft_magnitude_i ** 2)` for all available
  `fft_magnitude_X` columns. If absent, `NaN`.
